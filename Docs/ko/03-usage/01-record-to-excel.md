# 3.1 Excel 작업하기

> 이 챕터는 **데이터 작업자 (기획자)** 를 위한 안내입니다. Excel 시트가 이미 자리잡았거나 새로 만들 때, 어떤 모양으로 채우면 Sdp 가 받아 가는지 정리합니다. C# 쪽에서 Record 를 작성하는 흐름은 [3.3 첫 Record 정의하기](./03-first-record.md) 에서 다룹니다.

데이터 작업자는 자신이 다루기 편한 모양대로 시트를 구성하고, 프로그래머는 그 시트를 보고 필요한 컬럼만 골라 Record 로 표현합니다. 이 챕터는 시트 쪽에서 자주 마주치는 패턴을 정리합니다.

## 파일과 시트 이름

C# Record 에 `[StaticDataRecord("파일이름", "시트이름")]` 이 붙어 있고, 추출기는 이 정보로 Excel 파일과 시트를 찾습니다. 파일 이름과 시트 이름은 프로그래머가 Excel 파일과 시트의 현재 상태를 보고 그대로 적어 가면 되므로, 데이터 작업자가 미리 합의해 줘야 하는 정보는 아닙니다 — 시트 이름 표기 (대소문자 포함) 만 그대로 유지하면 됩니다.

추출 결과 CSV 는 `{파일}.{시트}.csv` 규칙으로 만들어집니다. 예: `GameItems.xlsx` 의 `Items` 시트 → `GameItems.Items.csv`.

## 헤더가 시작하는 셀

Sdp 의 추출기는 헤더가 어디서 시작하는지를 `--start-cell` 옵션으로 받습니다. 옵션을 생략하면 `A1` 부터 헤더가 시작한다고 가정합니다. **어느 셀에서 시작해도 무관** 합니다.

- 옵션이 가리키는 셀 = **첫 번째 컬럼의 헤더 이름이 들어가는 자리**
- 그 다음 행 = **첫 번째 데이터 행이 시작하는 자리**

위쪽에 메모나 타이틀을 두고 싶으면 그만큼 내려간 주소를 옵션으로 넘기면 됩니다 (예: `--start-cell B3`).

```bash
ExcelColumnExtractor.exe ^
  --record-path ./Records ^
  --excel-path ./Excels ^
  --output-path ./Csv ^
  --start-cell B3
```

자동화에 들어가면 한 프로젝트 안에서는 시작 위치를 하나로 합의해 두는 편이 단순합니다.

## 단순한 시트 — Item 예제

다음처럼 채워진 시트가 있다고 합시다.

|       | **A**  | **B**    | **C**   | **D**        |
|-------|--------|----------|---------|--------------|
| **1** | Id     | Name     | Price   | Category     |
| **2** | 1      | Potion   | 100     | Consumable   |
| **3** | 2      | Sword    | 5000    | Weapon       |
| **4** | 3      | Shield   | 4000    | Armor        |

- **1행이 헤더**, **2행부터 데이터** 입니다.
- `Category` 열에는 enum 멤버 이름 (`Consumable`, `Weapon`, `Armor`) 을 그대로 적습니다. 정수가 아닙니다.

프로그래머는 이 시트를 보고 필요한 컬럼만 골라 Record 를 작성합니다.

```csharp
[StaticDataRecord("GameItems", "Items")]
public sealed record ItemRecord(
    int Id,
    string Name,
    int Price,
    ItemCategory Category);
```

헤더 이름과 Record 파라미터 이름은 정확한 표기 (대소문자 포함) 가 일치해야 합니다.

## 참고 컬럼은 그대로 둬도 된다

기획자 입장에서 비고나 메모처럼 시트 안에서만 보고 싶은 컬럼이 자주 필요합니다. **Record 가 모르는 컬럼은 추출 시 무시되므로 그대로 두어도 됩니다.**

|       | **A**  | **B**    | **C**     | **D**   | **E**        |
|-------|--------|----------|-----------|---------|--------------|
| **1** | Id     | Name     | Memo      | Price   | Category     |
| **2** | 1      | Potion   | 회복 아이템 | 100     | Consumable   |
| **3** | 2      | Sword    | 기본 검    | 5000    | Weapon       |
| **4** | 3      | Shield   | 기본 방패  | 4000    | Armor        |

`Memo` 컬럼은 Record 에 없으므로 추출되지 않습니다. 데이터 작업자끼리 공유할 메모를 시트에 그대로 남길 수 있다는 의미입니다.

반대로 Record 가 요구하는 컬럼이 시트에 **없으면 추출이 실패합니다.**

## 문자열 셀의 쉼표, 줄바꿈, 큰따옴표

문자열 셀에는 쉼표, 줄바꿈 (Excel 셀 안에서 `Alt+Enter`), 큰따옴표를 그대로 넣을 수 있습니다. 추출기가 CSV 로 내보낼 때 표준 CSV 규약대로 큰따옴표로 감싸고 셀 안의 `"` 는 `""` 로 이스케이프합니다. C# 측에서도 그대로 원본 문자열로 복원됩니다.

|       | **A**  | **B**    | **C**                                     |
|-------|--------|----------|-------------------------------------------|
| **1** | Id     | Name     | Description                               |
| **2** | 1      | Potion   | HP, MP를 회복                              |
| **3** | 2      | Letter   | (셀 안 줄바꿈)<br>"안녕"이라고<br>적혀 있다 |

추출 결과 CSV 에서는 두 번째 행 `Description` 셀이 `"HP, MP를 회복"`, 세 번째 행은 큰따옴표 두 번 + 실제 줄바꿈 문자가 포함된 큰따옴표로 감싼 값으로 기록되며, C# Record 에는 시트에 보이는 그대로 들어옵니다.

## enum 컬럼

Record 의 enum 파라미터는 셀에 **enum 멤버 이름** 을 그대로 적습니다.

```csharp
public enum ItemCategory
{
    Consumable,
    Weapon,
    Armor,
}
```

|       | **A**  | **B**    | **C**        |
|-------|--------|----------|--------------|
| **1** | Id     | Name     | Category     |
| **2** | 1      | Potion   | Consumable   |
| **3** | 2      | Sword    | Weapon       |

매칭은 **대소문자를 구별** 합니다. `consumable`, `CONSUMABLE` 은 모두 로드 실패가 됩니다. 정의되지 않은 이름 (`Magic` 등) 도 마찬가지입니다.

## 같은 타입의 값이 여러 개 — 컬렉션

한 행에 같은 종류의 값을 여러 개 담아야 할 때가 있습니다. 예를 들어 한 아이템이 보유한 태그 목록처럼 말입니다. 두 가지 표현 방식 중에 데이터의 성격에 맞는 쪽을 고릅니다.

### 방식 A — 한 셀에 구분자로 묶기

태그가 그저 라벨이고 개별 값 검토가 필요 없다면, **한 셀에 쉼표로 묶어 두는** 방식이 가벼워서 좋습니다.

|       | **A**  | **B**    | **C**            |
|-------|--------|----------|------------------|
| **1** | Id     | Name     | Tags             |
| **2** | 1      | Potion   | heal,consumable  |
| **3** | 2      | Sword    | melee,iron       |
| **4** | 3      | Shield   | defense,iron     |

프로그래머 측에서는 `[SingleColumnCollection(",")]` 으로 받습니다.

```csharp
[StaticDataRecord("GameItems", "Items")]
public sealed record ItemRecord(
    int Id,
    string Name,
    [SingleColumnCollection(",")] ImmutableArray<string> Tags);
```

장점은 한 줄이 짧다는 것. 단점은 **개별 태그를 셀 단위로 비교하거나 정렬할 수 없다는 점** 입니다.

### 방식 B — 여러 셀로 펼치기

태그 하나하나가 검토 대상이거나, 셀 단위로 정렬, 필터링하고 싶다면 **여러 셀로 펼치는** 방식이 유리합니다.

|       | **A**  | **B**    | **C**     | **D**        | **E**     |
|-------|--------|----------|-----------|--------------|-----------|
| **1** | Id     | Name     | Tags[0]   | Tags[1]      | Tags[2]   |
| **2** | 1      | Potion   | heal      | consumable   | small     |
| **3** | 2      | Sword    | melee     | iron         | starter   |
| **4** | 3      | Shield   | defense   | iron         | starter   |

프로그래머 측에서는 `[Length(n)]` 으로 받습니다.

```csharp
[StaticDataRecord("GameItems", "Items")]
public sealed record ItemRecord(
    int Id,
    string Name,
    [Length(3)] ImmutableArray<string> Tags);
```

이렇게 두면 Excel 의 필터, 정렬 기능이 각 태그를 독립적으로 다룰 수 있습니다. **사용하지 않는 자리를 비워 두고 싶다** 면 어떤 토큰 (`"-"`, `"NULL"` 등) 을 빈 값으로 약속할지 한 번 합의해 두면 됩니다. 이때 Record 의 해당 원소는 nullable 로 두고 (`ImmutableArray<string?>`), 그 토큰을 `[NullString]` 으로 지정합니다.

선택 기준을 한 줄로 줄이면 다음과 같습니다.

- **셀 단위로 다룰 일이 없다** → 한 셀에 묶기 (`SingleColumnCollection`)
- **셀 단위로 비교, 필터, 검토한다** → 여러 셀로 펼치기 (`Length`)

## 객체 여러 개 — 객체의 배열

태그처럼 단순한 값이 아니라, **여러 필드를 가진 묶음** 이 한 행에 여러 개 들어가는 경우가 있습니다. 예를 들어 학생 한 명이 여러 과목의 성적을 가지는 시트입니다. 데이터 작업자 입장에서 헤더는 알아보기 쉬운 기획 이름으로 먼저 적어 두는 편이 자연스럽습니다.

|       | **A** | **B**   | **C**        | **D**     | **E**          | **F**       | **G**         | **H**      |
|-------|-------|---------|--------------|-----------|----------------|-------------|---------------|------------|
| **1** | Id    | Name    | 수학과목     | 수학점수  | 영어과목       | 영어점수    | 과학과목      | 과학점수   |
| **2** | 1     | Alice   | Math         | 90        | English        | 85          | Science       | 88         |
| **3** | 2     | Bob     | Math         | 70        | English        | 95          | Science       | 75         |
| **4** | 3     | Carol   | Math         | 80        | English        | 80          | Science       | 90         |

그러나 자동 추출이 동작하려면 헤더가 정해진 표준 형태여야 합니다. 위 시트의 표준 헤더는 다음과 같이 길어집니다.

|       | **A** | **B**   | **C**                 | **D**               | **E**                 | **F**               | **G**                 | **H**               |
|-------|-------|---------|-----------------------|---------------------|-----------------------|---------------------|-----------------------|---------------------|
| **1** | Id    | Name    | Subjects[0].Subject   | Subjects[0].Score   | Subjects[1].Subject   | Subjects[1].Score   | Subjects[2].Subject   | Subjects[2].Score   |

이 표준 헤더를 손으로 맞추는 일은 사실상 불가능에 가깝습니다. 이 부분을 자동으로 처리해 주는 것이 **표준 헤더 생성기** 입니다. 다음 챕터에서 이어 가겠습니다.
