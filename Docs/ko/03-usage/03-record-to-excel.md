# 3.3 Record를 먼저 쓰고 Excel 작업하기

앞서 3.1 \~ 3.2는 Excel이 이미 존재한다는 전제였습니다. 이번 장은 반대입니다. C# Record 가 먼저 확정되었고, 이제 그것에 맞춰 Excel 을 만드는 상황입니다.

이 흐름이 중요한 이유는, Excel 담당자가 빈 시트 위에 어디서부터 어떻게 채워야 하는지 모르기 때문입니다. 규칙을 정리해 둡니다.

## 헤더 시작 셀은 추출기 옵션으로 지정한다

Sdp 의 추출기는 **헤더가 어디서 시작하는지를 `--start-cell` 옵션으로 받습니다.** 옵션을 생략하면 `A1` 부터 헤더가 시작한다고 가정합니다.

- `--start-cell` 이 가리키는 셀 = 첫 번째 컬럼의 헤더 이름
- 그 다음 행 = 첫 번째 데이터 행의 첫 번째 값

가장 단순한 형태는 `A1` 부터 헤더, `A2` 부터 데이터를 두는 식이며, 이 경우에는 옵션을 생략해도 됩니다. 빈 줄이나 타이틀 행, 메모를 위에 두고 싶다면 그만큼 내려간 주소를 옵션 값으로 넘기면 됩니다 (예: `--start-cell B3`).

```bash
ExcelColumnExtractor.exe ^
  --record-path ./Records ^
  --excel-path ./Excels ^
  --output-path ./Csv ^
  --start-cell B3
```

추출기는 시트별로 같은 시작 셀을 사용하므로, 같은 배치에 함께 추출되는 시트들은 동일한 시작 위치 규칙을 공유해야 합니다. 시트마다 시작 위치를 달리하고 싶다면 추출 호출을 분리합니다.

## 파일·시트 이름 규칙

`[StaticDataRecord("Items", "Items")]` 를 붙인 Record라면, 추출기는 다음을 찾습니다.

- `Items.xlsx` 파일 (또는 `.xls`)
- 그 안의 `Items` 시트

파일 이름은 `ExcelColumnExtractor` 에게 `--excel-path` 로 디렉터리를 주면 해당 디렉터리에서 `Items.xlsx` 를 찾는 식입니다. 시트는 지정된 이름이 정확히 일치해야 합니다.

## 컬럼 이름

파라미터 이름이 곧 헤더 이름입니다. 3.1 의 `Item` 을 예로 들면 시작 셀부터 오른쪽으로 네 칸 (예: `A1` \~ `D1`) 에 다음과 같이 적어 두면 됩니다.

|A|B|C|D|
|-|-|-|-|
|Id|Name|Price|Category|

### 컬렉션이 있을 때

컬렉션을 쓰려면 Record 에서 `[Length(n)]` 으로 크기를 지정하고, Excel 에는 확장된 헤더를 둡니다. 예를 들어 `[Length(3)] ImmutableArray<int> Scores` 라면 Excel 헤더는 다음과 같이 세 개로 확장됩니다.

|Scores[0]|Scores[1]|Scores[2]|
|-|-|-|

헤더를 이렇게 일일이 손으로 맞추기는 번거로우므로, 다음 장의 [표준 헤더 생성기](./04-header-generator.md) 가 Record 로부터 이 헤더를 자동으로 뽑아 줍니다.

## 참고 컬럼은 그대로 두어도 된다

기획자가 비고·메모 등 시트 내부에서만 쓰이는 참고 컬럼을 추가하고 싶을 때가 있습니다. **Record 가 모르는 컬럼은 추출 시 무시되므로 그대로 둬도 됩니다.** Record 가 요구하는 컬럼만 정확한 이름으로 헤더에 들어 있으면 충분합니다.

반대로 Record 가 요구하는 컬럼이 시트에 **없으면** 추출이 실패합니다. 헤더 이름은 정확한 표기 (대소문자 포함) 가 일치해야 합니다.
